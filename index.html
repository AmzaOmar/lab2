02

Concept — La Navbar Bootstrap

Qu'est-ce qu'une Navbar ?

La navbar est la barre de navigation en haut d'une page. Bootstrap fournit une navbar responsive qui se transforme automatiquement en hamburger sur mobile (breakpoint configurable).

Anatomie de la Navbar

Voici la structure obligatoire :

  • .navbar → conteneur principal
  • .navbar-brand → logo / nom du site
  • .navbar-toggler → bouton hamburger (visible sur mobile)
  • .navbar-collapse → conteneur du menu (se cache sur mobile)
  • .navbar-nav → liste des liens de navigation
💡 Mobile-First : Sur les petits écrans, le menu est caché par défaut. En cliquant sur ☰ (hamburger), il s'ouvre. Sur desktop, le menu est toujours visible.
HTML — Structure de base 
<nav class="navbar navbar-expand-lg navbar-dark bg-dark">
  <div class="container">
    <!-- Logo -->
    <a class="navbar-brand fw-bold" href="#">🚀 DevAgency</a>

    <!-- Bouton hamburger (visible que sur mobile) -->
    <button class="navbar-toggler" type="button"
            data-bs-toggle="collapse"
            data-bs-target="#navMenu">
      <span class="navbar-toggler-icon"></span>
    </button>

    <!-- Menu (se cache sur mobile) -->
    <div class="collapse navbar-collapse" id="navMenu">
      <ul class="navbar-nav ms-auto">
        <li class="nav-item">
          <a class="nav-link active" href="#">Accueil</a>
        </li>
        <li class="nav-item">
          <a class="nav-link" href="#services">Services</a>
        </li>
      </ul>
    </div>
  </div>
</nav>
⚠️ Important : La valeur de data-bs-target (ici "#navMenu") doit correspondre exactement à l'id du .collapse. C'est le lien entre le bouton et le menu !
03

Concept — Section Héro

Qu'est-ce qu'une section Héro ?

C'est la première section visible après la navbar. Elle doit capter l'attention avec :

  • Un grand titre (h1 ou display-*)
  • Un sous-titre (lead)
  • Un ou plusieurs boutons d'action (CTA = Call To Action)
  • Éventuellement : une image, un dégradé, des icônes

Structure & Classes Bootstrap

Voici les classes clés :

  • .display-4 / .display-5 → très grands titres
  • .lead → paragraphe mis en avant (1.25rem)
  • .btn btn-primary / btn-outline-light → boutons d'action
  • .fw-bold → gras (font-weight: bold)
  • .text-center → centrer le texte
  • .bg-dark → fond sombre
  • .py-5 → padding vertical

05

Explications — Ligne par Ligne

🍔 Le Navbar Toggler

data-bs-toggle="collapse" data-bs-target="#navMenu"

Ces deux attributs disent à Bootstrap : "Quand on clique sur ce bouton, basculez la visibilité de l'élément avec id='navMenu'".

❗ Si l'id du .collapse est "#navMenu", alors data-bs-target DOIT être "#navMenu" aussi !

📱 fixed-top

La classe .fixed-top colle la navbar en haut lors du scroll. C'est pourquoi on ajoute .mt-5 à la section suivante — pour éviter que la navbar cache le contenu.

📐 align-items-center

Aligne verticalement le contenu du row. Ici, le texte et l'image de la héro sont centrés vertiquement l'un par rapport à l'autre.

🎨 display-5 & lead

.display-5 crée un très grand titre (3rem environ). .lead agrandit un paragraphe pour le mettre en avant (1.25rem).

🔘 d-flex & gap-3

.d-flex transforme un div en flexbox. .gap-3 ajoute de l'espace entre les boutons. .flex-wrap permet aux boutons de passer à la ligne sur mobile.

Comment faire — Étape par étape

01

Ouvrir votre Lab 1

Ouvrez le fichier index.html du Lab 1 dans VS Code ou Notepad++.

02

Remplacer le code

Sélectionnez tout le contenu <body> et remplacez-le par le code complet du Lab 2 ci-dessus.

03

Sauvegarder & tester

Sauvegardez avec Ctrl+S. Ouvrez la page dans Chrome. La navbar doit être en haut, la héro en dessous.

04

Tester le hamburger

Appuyez sur F12. Cliquez sur l'icône mobile pour réduire la fenêtre à 375px. Vous devez voir ☰ à la place du menu.

05

Cliquer sur le hamburger

Cliquez sur ☰. Le menu doit s'ouvrir / se fermer. Si c'est le cas, Lab 2 est terminé !

07

Exercices — Si vous avez terminé avant les autres

A

Changer les couleurs

Changez bg-dark en bg-primary pour la navbar. Changez aussi le héro section de bg-dark en bg-secondary. Testez avec différentes couleurs !

B

Ajouter des liens

Ajoutez 2 nouveaux liens au menu de la navbar : "À propos" et "Portfolio". Vérifiez qu'ils s'affichent bien sur desktop et se cachent bien sur mobile.

C

Modifier le breakpoint

Changez navbar-expand-lg en navbar-expand-md. Observez : maintenant le hamburger apparaît plus tôt (à partir de 768px au lieu de 992px).
